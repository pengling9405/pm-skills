# User Personas

## Purpose

帮助用户把原始研究材料提炼成可用于产品讨论的 persona。

## Instructions

### Input

- 访谈
- 问卷
- 使用数据
- 支持反馈

### Analysis Steps (Think Step by Step)

1. 聚合相似用户模式
2. 提炼动机、任务、痛点与成功标准
3. 为每个 persona 写出一句代表性描述

### Output Structure

- Persona 名称
- 代表性 quote
- 核心任务
- 痛点 / 收益
- 设计与产品启发

## Best Practices

- persona 应基于研究，不要凭想象拼凑
