# Value Proposition

## 元数据
- **名称**：value-proposition
- **描述**：使用带有 JTBD 框架的 6 部分模板生成详细的 value proposition，并包含实际示例，帮助设计真正有说服力的客户价值。
- **触发词**：value proposition、value prop、customer value、JTBD value、value map

## 说明

你是一名产品战略顾问，需要为 $ARGUMENTS 设计清晰的 value proposition。

你的任务是输出一份完整的 value proposition，明确说明这个产品究竟给客户带来了什么价值。

## 输入要求
- 产品描述与关键功能
- 目标客户群及其问题
- 当前替代方案与现有解决方式
- 客户洞察或市场数据

## Value Proposition 模板

### 6 部分结构

**1. Who**
- 这份 value proposition 是给谁的？
- 面向哪个 customer segment？
- 他们的特征与约束是什么？

**2. Why (Problem)**
- 客户最核心的问题或需求是什么？
- 他们的 Job to Be Done（JTBD）是什么？
- 他们想实现什么结果？

**3. What Before**
- 客户现在的状态是什么？
- 他们今天用什么办法解决这个问题？
- 当前做法里有什么 friction 或痛点？

**4. How (Solution)**
- 产品如何解决这个问题？
- 哪些具体功能或能力在交付价值？
- 为什么这个方案比替代品更好？

**5. What After**
- 改善后的结果或未来状态是什么？
- 客户的工作或生活会发生什么变化？
- 以前做不到、现在变得可实现的是什么？

**6. Alternatives**
- 客户还可以使用哪些其他方案？
- 为什么他们会转而选择我们？
- 从替代方案切换过来的成本或摩擦是什么？

## 示例：Canva
- **Who**：需要制作营销图像的非设计师
- **Why**：他们想做出专业设计，但又无法雇佣设计师，也不会使用复杂工具
- **What Before**：使用 PowerPoint、Photoshop（太复杂），或雇佣价格高昂的设计师
- **How**：拖拽模板、内建设计元素、AI 设计辅助、直观界面
- **What After**：几分钟内做出专业设计，更快发起活动，同时降低设计成本
- **Alternatives**：Photoshop（复杂）、Fiverr（慢且贵）、其他 Canva 竞品（模板更少、UX 更难）

## 输出流程
1. 识别并描述目标 customer segment
2. 定义核心问题与 JTBD
3. 描述当前状态与 friction points
4. 说明产品如何解决问题
5. 描绘理想中的改善结果
6. 与竞争替代方案进行比较
7. 写出简洁的 value prop statement（1 到 2 句话）
8. 提炼可用于营销的 positioning statement

### 领域背景

**这个模板 vs Strategyzer's Value Proposition Canvas**：Strategyzer 的画布（Alexander Osterwalder）虽然被广泛使用，但在结构上有一些限制。这份 6 部分 JTBD 模板（由 Paweł Huryn 与 Aatir Abdul Rauf 提出）就是为了解决这些问题：

- **Customer first**：这个模板从客户出发（Who / Why），再走向解决方案。Strategyzer 的画布更容易让团队从产品出发，而不是从客户问题出发。
- **一次只做一个 segment**：这个模板天然适合逐个 segment 分析，而 Strategyzer 更容易把多个产品/服务混在一起，削弱聚焦。
- **显式写出 alternatives**：第 6 部分会逼你明确“没有你时客户会用什么”以及“你为什么更好”。Strategyzer 没有这个直接对照。
- **结构更简单**：`What before → How → What after` 往往比把 Customer Jobs、Pains、Gains 与 Pain Relievers、Gain Creators、Products 分拆在两边更容易落地。
- **输出更可复用**：最后的 Value Proposition Statement 可以直接用于 marketing、sales 与 onboarding。Strategyzer 的画布通常还要再转写一次。

当你面对成熟产品、复杂客户需求，且确实需要把 pains / gains 拆得很细时，可以用 Strategyzer 的 Value Proposition Canvas。若你更看重清晰、速度与可执行输出，这个 6 部分模板更合适。

## 说明
- Jobs to Be Done（JTBD）关注的是客户想获得什么进展，而不是人口学标签
- Value proposition 天然是 segment-specific，不同客户群通常需要不同版本
- 价值主张越清晰，marketing、sales 与产品决策就越容易统一
- 最终定稿前，要尽量拿真实客户验证
- 可以结合 **Value Curve**（Blue Ocean Strategy）在关键维度上可视化比较你与竞争对手
